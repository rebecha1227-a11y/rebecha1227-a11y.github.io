# ChatBox 人机恋 Web App

一个偏 iMessage / 微信风格的恋爱聊天网页应用（单文件前端）。

## 这是个什么

你可以把它当成「恋爱聊天模拟器 + 关系纪念 + 朋友圈动态 + 数据管理」：

- 聊天页：看会话列表、新建聊天、全局搜索入口
- 列表页：联系人分组、搜索、创建/编辑角色
- 动态页：发动态、留言板、封面图
- 我页：聊天外观、云同步、记忆管理、收藏、导出、纪念日、API 设置

## 本地运行

这个项目是纯前端静态文件，不需要先编译。

### 方式 A（推荐）

在项目目录执行：

```bash
python3 -m http.server 3456 -d .
```

打开：

```text
http://localhost:3456
```

### 方式 B

直接双击 `index.html` 也能打开，但部分浏览器策略下功能可能受限，建议优先用方式 A。

## 我实测通过的功能

以下是本次自动化体验里通过的核心流程：

1. 页面可正常打开（title: `Jinger's Palace`）
2. 底部 4 个页签可切换：消息 / 列表 / 动态 / 我
3. 列表页联系人搜索框可输入
4. 动态页 `+ 发动态` 入口可点击触发
5. 消息页 `✏️` 新建聊天入口可点击触发
6. 聊天输入框逻辑正常：未进入具体会话前默认不可见

## 当前已知现象（不是崩溃）

1. 从动态页打开“发动态”弹层后，如果不先关闭弹层，再点底部页签会被遮罩层拦截（页面代码里是 `compose-modal` 覆盖在上面）。
2. 聊天发送要先进入一个具体联系人会话；只停在消息列表时，输入框不可见属于预期行为。

## 主要文件

- `index.html`：主页面 + 样式 + 交互逻辑（核心都在这里）
- `imessage-adapted.css`：额外样式文件
- `supabase_schema.sql`：云端数据结构示例
- `SUPABASE_入门.md`：Supabase 接入说明

## 建议的使用顺序

1. 先去「列表」创建或选择联系人
2. 再回「消息」进入会话聊天
3. 需要分享生活就去「动态」发内容
4. 需要个性化和数据管理去「我」页设置
